# C++ Module 09 確認観点チェックリスト

---

## 共通ルール

- [ ] `c++` でコンパイルし、`-Wall -Wextra -Werror` フラグが通る
- [ ] `-std=c++98` フラグを追加してもコンパイルが通る
- [ ] `*printf()` / `*alloc()` / `free()` を使っていない
- [ ] `using namespace` / `friend` キーワードを使っていない
- [ ] クラスは Orthodox Canonical Form（デフォルトコンストラクタ・コピーコンストラクタ・代入演算子・デストラクタ）を実装している
- [ ] ヘッダファイルに関数の実装を書いていない（テンプレートを除く）
- [ ] すべてのヘッダにインクルードガードがある
- [ ] メモリリークがない
- [ ] Makefile が `$(NAME)` / `all` / `clean` / `fclean` / `re` を持つ
- [ ] Makefile が不必要な relink をしない

## モジュール固有ルール

- [ ] 各 Exercise で STL コンテナを少なくとも1つ使っている
- [ ] Exercise 02 は異なる STL コンテナを2つ使っている
- [ ] ex00 で使ったコンテナを ex01 で使っていない
- [ ] ex01 で使ったコンテナを ex02 で使っていない（ex00 で使ったものも不可）
- [ ] dequeは内部で、stackを使っているためそれを避けるためのmutantを実施していること

## Exercise 00: Bitcoin Exchange

### 基本動作

- [ ] バイナリ名が `btc` である
- [ ] 引数なしで実行したときに `Error: could not open file.` を出力する
- [ ] 引数にファイルを受け取り、正しく処理できる

### 入力ファイルのパース

- [ ] `date | value` 形式の各行を正しくパースできる
- [ ] 日付が `YYYY-MM-DD` 形式で正しく検証される
- [ ] 不正な日付（例: `2001-42-42`）に対して `Error: bad input => <line>` を出力する
- [ ] 区切り文字 `|` がない行を不正入力として扱う

### 値のバリデーション

- [ ] 値が負のとき `Error: not a positive number.` を出力する
- [ ] 値が 1000 を超えるとき `Error: too large a number.` を出力する
- [ ] 値が float または正の整数として正しく解釈される

### データベースの検索

- [ ] CSV データベースを正しく読み込んでいる
- [ ] 入力日付がDBに存在する場合、その日のレートを使う
- [ ] 入力日付がDBに存在しない場合、それより前で最も近い日付のレートを使う（上の日付ではなく下の日付）
- [ ] DBより古い日付（最古より前）の扱いが適切である

### 出力

- [ ] 正常時の出力形式が `YYYY-MM-DD => value = result` である
- [ ] 浮動小数点の計算結果が正しい

### コンテナ

- [ ] `std::map` 相当のコンテナ（日付→レートの対応）を使っている

## Exercise 01: Reverse Polish Notation

### 基本動作

- [ ] バイナリ名が `RPN` である
- [ ] コマンドライン引数として逆ポーランド記法の式を受け取る
- [ ] 正しい計算結果を標準出力に出力する
- [ ] エラー時は標準エラーにエラーメッセージを出力する

### サポートする演算子

- [ ] `+`（加算）が動作する
- [ ] `-`（減算）が動作する
- [ ] `/`（除算）が動作する
- [ ] `*`（乗算）が動作する

### エラーハンドリング

- [ ] 括弧 `(` `)` を含む式に対して `Error` を出力する
- [ ] 演算子が足りない（オペランドが余る）場合にエラーになる
- [ ] オペランドが足りない（スタックが枯渇する）場合にエラーになる
- [ ] 10以上の数値がトークンとして渡された場合の扱いが適切である
- [ ] 0除算の扱いが適切である

### 計算仕様

- [ ] 引数の数値は常に 10 未満（1桁）だが、計算途中・結果は10以上になりうる
- [ ] 小数・括弧は扱わなくてよい

### 動作確認例

```
./RPN "8 9 * 9 - 9 - 9 - 4 - 1 +"  => 42
./RPN "7 7 * 7 -"                    => 42
./RPN "1 2 * 2 / 2 * 2 4 - +"       => 0
./RPN "(1 + 1)"                      => Error
```

### コンテナ

- [ ] スタックとして機能するコンテナ（`std::stack` 相当）を使っている
- [ ] ex00 で使ったコンテナと異なるものを使っている

## Exercise 02: PmergeMe

### 基本動作

- [ ] バイナリ名が `PmergeMe` である
- [ ] コマンドライン引数として正の整数列を受け取る
- [ ] 負の数が渡されたとき `Error` を出力する
- [ ] 少なくとも 3000 個の整数を処理できる

### アルゴリズム

- [ ] **Ford-Johnson アルゴリズム**（Merge-Insert Sort）を実装している
- [ ] 汎用関数に頼らず、各コンテナに対して個別に実装している
- [ ] ソート結果が正しい

### 出力フォーマット

- [ ] 1行目: ソート前の数列を表示する（例: `Before: 3 5 9 7 4`）
- [ ] 2行目: ソート後の数列を表示する（例: `After:  3 4 5 7 9`）
- [ ] 3行目: 1つ目のコンテナで要素数と処理時間を表示する
- [ ] 4行目: 2つ目のコンテナで要素数と処理時間を表示する

### 時間計測

- [ ] ソートと入力処理の両方を含む全体の時間を計測している
- [ ] 2つのコンテナの差がわかる精度で時間を表示している（マイクロ秒推奨）

### コンテナ

- [ ] 異なる2種類の STL コンテナを使っている（例: `std::vector` と `std::deque`）
- [ ] ex00・ex01 で使ったコンテナを使っていない

### 重複値

- [ ] 重複した整数の扱い方針が決まっている（エラーにするか許容するか）
