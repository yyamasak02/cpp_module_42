### マージインサートソート
- page: p184

- **マージ挿入法**  
    上記の方法の優れた一般化が、Lester Ford, Jr.とSelmer Johnsonによって発見されました。  
    この方法はマージの側面と挿入の側面の両方を含んでいるため、私たちはこれを「マージ挿入法」と呼ぶことにします。

    例として、21個の要素をソートする問題を考えてみましょう。  
    まず10個のペア K₁:K₂、K₃:K₄、...、K₁₉:K₂₀ を比較することから始めます。  
    次に、各ペアの大きい方の要素10個を、マージ挿入法を使ってソートします。  
    その結果、以下のような配置が得られます。

    - ![画像1](images/p184_1.png)

    (5)に類似した配置。  
    次のステップは **b₃** を {b₁, a₁, a₂} の中に挿入し、その後 **b₂** を a₂より小さい他の要素の中に挿入します。  
    これにより以下の配置に到達します。

    - ![画像2](images/p184_2.png)

    上の行の要素を**メインチェーン**と呼ぶことにしましょう。  
    **b₅** をメインチェーンの適切な位置に挿入することができ、これには3回の比較が必要です  
    （最初にc₄と比較し、次にc₂またはc₆などと比較します）。  

    その後、**b₄**をさらに3回のステップでメインチェーンに移動でき、以下の配置になります。

    - ![画像3](images/p184_3.png)

    次のステップが重要です。何をすべきか明確でしょうか？  
    **b₁₁**（b₇ではなく）をメインチェーンに挿入します。これにはわずか4回の比較しか必要ありません。  
    その後、**b₁₀、b₉、b₈、b₇、b₆**（この順序で）もメインチェーンの適切な位置に挿入でき、  
    それぞれ最大4回の比較で済みます。

    ここで行われる比較を注意深く数えると、21個の要素が最大で  
    ```
    10 + S(10) + 2 + 2 + 3 + 3 + 4 + 4 + 4 + 4 + 4 + 4 = 66回
    ```
    のステップでソートされたことがわかります。  
    **2⁶⁵ < 21! < 2⁶⁶** であることから、いかなる場合でも66回未満は不可能であることも分かります。  

    したがって、**S(21) = 66** です。（バイナリ挿入では74回の比較が必要だったでしょう。）

---

一般的に、n個の要素に対するマージ挿入法は以下のように進行します：

1. **ペア比較（i）**  
   ⌊n/2⌋個の互いに素なペアで要素同士の比較を行います。  
   （nが奇数の場合、1つの要素を除外します。）

2. **大きい方をソート（ii）**  
   ステップ(i)で見つかった⌊n/2⌋個の大きい方の数を、マージ挿入法でソートします。

3. **b群の挿入（iii）**  
    (1)のように要素を **a₁, a₂, ..., a₍ₙ/₂₎, b₁, b₂, ..., b₍ₙ/₂₎** と名付けます。  
    ここで、a₁ < a₂ ≤ ... ≤ a₍ₙ/₂₎ かつ 1 ≤ i ≤ ⌊n/2⌋ に対して bᵢ ≤ aᵢ です。  
    b₁ と a 達を **「メインチェーン」** と呼びます。  
    残りの b 達を、j > ⌊n/2⌋ のすべての bⱼ を除外して、以下の順序でバイナリ挿入を使ってメインチェーンに挿入します：
    ```
    b₃, b₂;
    b₅, b₄;
    b₁₁, b₁₀, ..., b₆;
    ...
    bₜₖ, bₜₖ₋₁, ..., bₜₖ₋₁₊₁; ...
    ```
    (11)に現れる数列 (t₁, t₂, t₃, t₄, ...) = (1, 3, 5, 11, ...) を、  
    bₜₖ, bₜₖ₋₁, ..., bₜₖ₋₁₊₁ のそれぞれが最大 k 回の比較でメインチェーンに挿入できるように定義したいと思います。
